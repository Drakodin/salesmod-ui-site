Web Components

But wait! Why this? Simple.

In the typical case, most component-based websites are locked to a specific framework that lets someone make their own. Not only that, it is not that easy to keep reusing code in just raw HTML projects.

That is where Web Components come in and why we chose to make a near universally supported library for all potential users. We know that reusing blocks is hard and web compoents solves this by allowing us to define our own blocks, power a "shadow" DOM which renders elements separately from the document itself, and allow us to use templates that can reused countless times.

Plus, they are not limited to just files. These are browser-end supports, not your framework. So whether you are using React or Liquid, these components can be added to your preference.
